Machine Learning vs. Deep Learning Case Study

  • Case Study
  • ML / DL
  • Technical Writing

Introduction

This artifact is a written case-study analysis comparing machine learning (ML) and deep learning (DL), produced for AIML-500 Machine Learning Fundamentals. Both are subfields of artificial intelligence, but they solve problems in different ways. Traditional machine learning relies on humans to hand-pick the important features in the data and then applies algorithms such as decision trees, support vector machines, or regression. Deep learning uses multi-layered neural networks that automatically discover the relevant features directly from raw data, at the cost of needing far more data and computing power.

Objective

The objective was to analyze, using real-world examples, when traditional machine learning is the better fit and when deep learning is the better fit. For each example I described a deployed real-world application, explained why the chosen approach suits the problem, and explained why the alternative approach would not.

Project Overview

The analysis studies two contrasting examples: one simple machine learning problem and one deep learning problem.

Example 1 (Simple Machine Learning): Customer Churn Prediction

Subscription businesses such as telecom carriers, streaming platforms, and SaaS companies need to know in advance which customers are likely to cancel so they can intervene with retention offers. A support vector machine (SVM) can be trained on clearly defined, structured features such as contract length, monthly charges, number of support calls, and tenure to flag at-risk customers. This is a proven, deployed application: the mobile operator Kcell reduced churn by 10.4% with a production churn model, and Nguyen et al. (2022) built a kernel SVM churn model that reached a 99% F1-score and 98.9% accuracy. Machine learning fits here because the data is structured, human-selected features work well, the datasets are small and cheap to train, and the results are interpretable. Deep learning is a poor fit because it is data hungry, adds unnecessary complexity and cost, and behaves like a black box.

Example 2 (Deep Learning): Image Recognition

In healthcare, deep learning is used to analyze medical images such as X-rays and MRIs to detect potentially cancerous lesions or tumors. A convolutional neural network (CNN) is trained on large collections of labeled scans and learns the subtle visual patterns that distinguish healthy tissue from suspicious growths, acting as a second set of eyes for a radiologist. Google Health and DeepMind built a breast-cancer screening system from an ensemble of three CNNs that reduced both false positives and false negatives and outperformed all six radiologists in an independent reader study (McKinney et al., 2020). Deep learning fits here because the data is unstructured and high-dimensional, features cannot realistically be hand-coded, the patterns are complex and nonlinear, and large datasets and GPUs are available. Simple machine learning is a poor fit because manual feature engineering on raw pixels is impractical and cannot capture spatial complexity.

Table 1 summarizes the key differences highlighted by these examples.

Aspect Simple Machine Learning (Churn Prediction) Deep Learning (Image Recognition)
Data type Structured / tabular Unstructured (raw images)
Feature extraction Manual (human-selected) Automatic (learned by the network)
Data volume needed Smaller datasets work well Requires large datasets
Computational cost Low; fast to train High; needs GPUs
Interpretability High (easier to explain) Low ("black box")
Best-fit example Predicting customer churn Detecting tumors in medical scans

Process

This was an individual assignment. I worked through the following steps:

  • Reviewed the lesson concepts on machine learning versus deep learning and their trade-offs.
  • Selected two contrasting real-world examples: one structured-data ML problem and one unstructured-data DL problem.
  • Researched deployed applications and peer-reviewed results to ground each example in evidence.
  • Analyzed why each approach fit its problem across data type, feature extraction, data volume, cost, and interpretability.
  • Summarized the comparison in a table and wrote the analysis in APA style with cited sources.

Tools & Technologies

  • Academic and technical research (peer-reviewed papers and industry case studies)
  • Comparative analysis across data type, cost, and interpretability
  • AI assistance for synthesizing the summary table, reviewed for accuracy

Value Proposition

This artifact demonstrates my ability to reason about machine learning design decisions, not just implement models. It shows that I can evaluate trade-offs between approaches, ground claims in real deployments and research, and communicate technical decisions clearly.

For those in software, data, and AI roles, this decision-making skill matters as much as coding: choosing the right approach for the data and constraints is often what makes a project succeed.

Key Learnings

The clearest takeaway is that neither approach is universally better. The right choice depends on the data type and volume, the available computing resources, and whether the results need to be easily explained. I learned to match the method to the problem: simple, interpretable models for structured data with well-understood features, and deep networks for large, unstructured data with complex patterns.

Researching real deployments also reinforced how much practical constraints such as cost, data availability, and interpretability shape real machine learning decisions, not just raw accuracy.
